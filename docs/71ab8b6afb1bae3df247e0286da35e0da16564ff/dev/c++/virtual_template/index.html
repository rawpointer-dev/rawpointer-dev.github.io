virtual 和 template

（一）、引言

首先，来看一个问题：

c++ 的 member function 能够既是 template 类型又是 virtual 类型呢？ 即：以下代码是否合法呢？

1
2
3
4
5
6
7

class Animal{
  public:
      template<typename T>
      virtual void make_sound(){
        //...
      }
};

显然，答案是不行。用一句话来描述就是虚函数表无法填。因为：在虚函数表中，我们为每个虚函数存储一个函数指针，当该虚函数有多个类型时，我们无法在这一个位置存储多个不同类型对应的函数指针。

（二）、引申

那么，我们再看另一个问题：

template class 中的 member function 是否能为 virtual 或者 pure virtual 呢？

答案是 yes。

原因是：和上一个例子不同的是在 template class 中，编译时首先会根据不同参数类型生成相应的 class ，后续的 virtual memeber function 是在该特定类型的 class 中的，那么 vtable 的填写就不成问题了。

（三）、注意点

继承 template class 时，需要指定 template 参数，例如：

1
2
3
4

template <typename T>
class DenseDataset final : public TypedDataset<T> {
    ...
}

而不能：

1
2
3
4

template <typename T>
class DenseDataset final : public TypedDataset {
    ...
}
